使用Gitolite搭建Git服务器

发表于 2015-05-13   |   分类于 服务器   |  

Git是我目前最喜欢的版本管理系统。对于开源项目来说，Github一定是最好的选择，但是Github上创建私有项目是要收费的，所以对于个人或公司的一些私有项目，可以考虑搭建一个自己的Git服务器或者找一些免费提供私有项目服务的公司。

我使用过的提供免费私有项目服务的有三家：

  • 开源中国(https://git.oschina.net)
  • GitLib(https://about.gitlab.com)
  • BitBucket(https://bitbucket.org)。

如果想自己尝试搭建Git服务器或者公司要求代码要放在自己的服务器上，Gitolite可以非常快速的构建一个轻量的Git服务器。Gitolite的源码托管在Github上，项目地址：https://github.com/sitaramc/gitolite，主页上也有非常详细的安装和使用说明。

一、安装需求

  • Unix或Linux操作系统
  • sh
  • git 1.6.6 以上
  • perl 5.8.8 以上
  • openssh 5.0以上

二、安装步骤

    先创建一个Git服务所依赖的系统用户，通常为git，并且后面不要使用该用户直接登录服务器。

    使用git用户执行以下命令下载源码进行安装

    在需要Git服务的另一台客户端电脑上使用 ssh-keygen 生成公钥，输入命令后一直回车即可。生成的公钥默认为$HOME/.ssh目录下的id_rsa.pub，将该文件上传至Git服务器即可。
（PS: scp $HOME/.ssh/id_rsa.pub 服务器用户@服务器IP地址:/服务器存放目录）

    回到服务器端并切换到Git用户，将改客户端上传上来的公钥设置为Git服务器的用户，Gitolite即安装成功，服务器端配置结束。

三、配置与使用

    Gitolite安装完成后，不要直接在服务器添加用户或仓库，所有的用户和权限都通过一个特殊的仓库gitolite-admin来设置，配置完成后提交修改即可。在客户端电脑克隆该仓库进行修改

gitolite-admin下有两个目录：

  • keydir：把用户的公钥添加到keydir目录下即可自动成为Gitolite用户，用户名为公钥文件名（如添加pharen.pub相当于创建了一个用户名为pharen的Git用户）
  • conf：目录下有一个名为gitolite.conf配置文件用来配置仓库以及用户权限

gitolite.conf默认已经配置了两个仓库，并且admin具有读写权限。

修改gitolite.conf，添加一个仓库foo，并设置用户权限

提交修改到服务器后，Gitolite会自动帮我们建好foo仓库，并分配好用户权限。

随后我们通过 git clone git@服务器IP:foo 可以克隆该仓库进行使用

在gitolite中我们还可以将多个用户组成用户组，然后统一给用户组分配权限。也可以对仓库的分支进行权限分配等，详细的使用可以参考官方文档。
