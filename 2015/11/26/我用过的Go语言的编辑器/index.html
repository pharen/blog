我用过的Go语言的编辑器

发表于 2015-11-26   |   分类于 编程   |  

Golang有gocode、goimports、godoc等代码补全格式化工具，使得Golang可以直接使用众多文本编辑器来进行开发，如Vim、Emacs、Atom、Sublime等。
而且代码补全和代码格式化功能确实非常完善，完全不输给Xcode、Android Studio等IDE的代码提示补全功能。

Go常用工具安装

Atom

我最初开发Go语言程序使用的是Atom，这个由Github团队开发的编辑器是完全免费的，并且支持插件扩展，可配置性非常高，同时Atom也支持 Windows、Mac、Linux三大平台。
Atom支持Golang语法高亮、代码补全等功能非常简单，只需要安装go-plus这个包即可。

用Atom开发了一段时间Golang，渐渐发现Atom启动速度比较慢，尤其是在公司那台还用着机械硬盘的iMac上，有时点击打开一个文件要等好几十秒，
而且我指定编辑某个文件还会连上次关闭的文件和目录都打开，感觉特别麻烦。

Sublime Text 3

因为Atom启动慢的问题，所以就换成了很多人推荐的Sublime。Sublime使用方式跟Atom的使用方式非常类似，所以转过来也不怎么费时间上手。
安装好Sublime后还需要安装Package Control才能安装包。

Visual Studio Code

Visual Studio Code是微软基于Atom核心做的一款代码编辑器，但是启动速度比Atom快了不少。因为解决了Atom启动慢的缺点，所以果断转到Visual Studio Code来开发。
安装好VSCode后，也需要安装插件来支持Go的各种功能，Atom和Sublime叫Package，而VSCode叫Extension。安装插件方法跟Atom和Sublime类似，插件地址https://marketplace.visualstudio.com/items/lukehoban.Go。
另外VSCode还支持Go语言的debug功能，不过需要额外安装delve，并且不支持Windows。Mac OS X的安装方法在https://github.com/derekparker/delve/wiki/Tips-&-Troubleshooting。

LiteIDE

LiteIDE是比较全面的开发Go语言的IDE，各方面都不错，只是我只是业余谢谢Go，普通的文本编辑器更适合我。
